# Proxmox VE（PVE）安装引导

[返回 README](../../README.md) | 中文

本文面向第一次在 **Proxmox VE（PVE）** 上使用 Landscape Mini 镜像的用户，按步骤从 0 开始引导安装。

## 先看这里

如果你只是想尽快装起来，**优先直接使用仓库 Release 里已经构建好的镜像**。

只有在以下场景下，才建议再去使用 [Custom Build 使用说明](./custom-build.md)：

- 你需要自定义网段
- 你需要修改 LAN / DHCP 参数
- 你需要自定义密码、账号或其他构建参数

推荐顺序：

1. 能直接用官方 Release 镜像时，优先直接安装
2. 如果需要自定义网络参数，再使用 `Custom Build`
3. 默认推荐选择 **带 Docker** 的构建
4. 优先选择 `output_formats=img,ova`
5. 在 PVE 中优先使用 **OVA 导入**
6. 如果 OVA 不方便，再使用 `.img` / `.img.gz` 手动导入

---

## 一、推荐构建方式

如果你是第一次使用，推荐在 `Custom Build` 中优先选择：

- `base_system=debian`
- `include_docker=true`
- `output_formats=img,ova`

推荐理由：

- `debian` 兼容性更稳，适合作为默认选择
- `include_docker=true` 更符合常见实际使用场景，避免后续再补装 Docker
- `img,ova` 会同时生成：
  - `.ova`：适合 PVE 一键导入
  - `.img`：适合手动导入和兜底排障

> 如果你不需要自定义网络参数，优先直接使用仓库 Release 中已经构建好的镜像即可。  
> 如果你走的是 `Custom Build`，请到你自己的 workflow 产物或固定 release 页面自行复制下载链接。

---

## 二、开始前准备

请先确认：

- 你有一个可用的 PVE 节点
- 你已经拿到了构建产物：
  - 推荐：`.ova`
  - 备用：`.img` 或 `.img.gz`
- 你知道要导入到哪个存储池
- 你可以登录 PVE Web 管理界面
- 如果走手动导入方式，你还可以通过 SSH 登录 PVE 宿主机

---

## 三、非常重要：多张网卡时网卡类型必须一致

如果你的虚拟机有多张网卡，请务必确认它们的**网卡类型一致**。

例如：

- 全部都是 `E1000`
- 或全部都是 `VirtIO`
- 或全部都是其他同一种类型

### 为什么要一致？

如果多张网卡类型不一致，可能会导致系统里的接口顺序混乱，例如：

- 本来希望 WAN 是 `eth0`
- 实际启动后 LAN / WAN 顺序对调
- 最终表现为：
  - `ip a` 里 `eth0` 没有分配 IP
  - 或 `eth0` / `eth1` 顺序和预期不一致

### 出现这个问题怎么处理？

处理方法：

1. 进入 PVE 虚拟机的**硬件**页面
2. 检查所有网卡的 model/type
3. 把它们改成**一致的类型**
4. **重启虚拟机**

> 当前不建议混用不同网卡类型，否则很容易出现接口顺序混乱。

---

## 四、方式 1：通过 URL 直接导入 OVA（推荐）

优先建议直接使用仓库官方 Release 中的 `.ova` 下载链接。

如果你使用的是 `Custom Build`，则请到你自己的 workflow 产物或固定 release 页面自行复制 `.ova` 下载链接。

### 第 1 步：确认存储池允许导入

进入：

**数据中心 -> 存储 -> 对应存储对象（例如 `local`）**

点击编辑，确认该存储的“内容”里已经勾选：

- `导入`
- `磁盘镜像`

这两个内容类型都需要开启，否则导入可能失败。

---

### 第 2 步：进入存储池的导入页面

进入：

**数据中心 -> 存储 -> 对应存储对象（例如 `local`） -> 内容**

找到 **从 URL 下载 / 导入** 的入口。

---

### 第 3 步：复制下载链接

如果你使用官方 Release，请直接复制对应 `.ova` 文件的下载链接。

然后把这个链接填入 PVE 的 URL 导入框中。

如果你使用的是 `Custom Build`：

- 请到你自己的 workflow 产物页面或固定 release 页面
- 自行找到对应的 `.ova` 文件
- 再复制下载链接填入 PVE

---

### 第 4 步：开始下载并导入

确认目标存储后，执行导入。

导入完成后，PVE 会得到可供虚拟机使用的镜像文件。

---

### 第 5 步：检查导入后的虚拟机配置

导入完成后，建议检查以下项目：

- 启动模式
- 磁盘控制器
- 网桥（bridge）绑定
- CPU 类型
- 网卡模型

说明：

- **老 CPU 推荐优先使用 `host` 模式**，兼容性通常更好
- 如果你希望 CPU type 使用 `host`，请在导入后手动设置
- PVE 当前不会稳定从 OVF/OVA 元数据中自动继承 `CPU type=host`

---

## 五、方式 2：手动导入 `.img` 或 `.img.gz`

如果你不想使用 OVA，或者 OVA 导入不方便，可以走 raw 镜像导入方式。

### 第 1 步：先在 PVE 中创建虚拟机

新建虚拟机时：

- 正常填写 VM 名称、ID 等信息
- **不要添加硬盘**
- 其他设置可以先按默认或你的需求填写

关键点：

- **创建虚拟机时先不要选择硬盘**
- 后续我们再把 `.img` 手动导入进去

---

### 第 2 步：把 `.img` 或 `.img.gz` 放到 PVE 宿主机上

你有两种常见方式。

#### 方式 A：在 PVE 宿主机上直接下载

如果你使用官方 Release，请先复制官方 `.img.gz` 下载链接，然后 SSH 登录 PVE 宿主机执行：

```bash
wget -O landscape-mini.img.gz "<官方 Release 下载链接>"
```

如果你使用的是 `Custom Build`，请到你自己的 workflow 产物或固定 release 页面自行复制 `.img` / `.img.gz` 下载链接，再执行同样的下载步骤。

如果下载到的是 `.img.gz`，请先解压：

```bash
gunzip -f landscape-mini.img.gz
```

解压后会得到：

- `landscape-mini.img`

如果你下载到的本来就是 `.img`，则不需要解压。

#### 方式 B：从本地手动上传

你也可以用以下方式上传到 PVE 宿主机任意目录：

- `scp`
- `rsync`
- SFTP 工具
- 其他你习惯的上传方式

例如：

```bash
scp landscape-mini.img root@<pve-host>:/root/
```

如果你上传的是 `.img.gz`，上传后也需要先在 PVE 宿主机上执行解压：

```bash
gunzip -f /root/landscape-mini.img.gz
```

---

### 第 3 步：导入磁盘到 PVE

登录 PVE 宿主机后执行：

```bash
qm importdisk <vmid> /path/to/landscape-mini.img <storage>
```

例如：

```bash
qm importdisk 101 /root/landscape-mini.img local-lvm
```

说明：

- `<vmid>`：虚拟机 ID
- `/path/to/landscape-mini.img`：镜像路径
- `<storage>`：目标存储池名称

---

### 第 4 步：在虚拟机硬件页面挂载导入后的磁盘

回到 PVE Web 页面：

**虚拟机 -> 硬件**

找到刚导入的磁盘，把它挂载到你要使用的位置（例如 `scsi0`、`sata0`）。

然后设置：

- 启动顺序
- 该磁盘为启动盘

---

## 六、首次启动后的检查

启动虚拟机后，建议先检查：

### 1. 网卡顺序是否正确

执行：

```bash
ip a
```

重点观察：

- `eth0` 是否拿到了预期的 IP
- `eth1` 是否对应你预期的另一张网卡
- WAN / LAN 是否和你的接线、bridge 配置一致

如果发现：

- `eth0` 没有 IP
- 或 `eth0` / `eth1` 顺序异常

请优先回到 PVE 检查多张网卡的类型是否一致。

---

## 七、磁盘扩容说明

Landscape Mini 支持在开机时自动扩展根分区到当前磁盘大小。

这意味着：

- **首次启动时**，会自动扩容到当前磁盘大小
- 后续如果你在 PVE 中把磁盘调大，**下次重启时**也会继续扩容

### 在 PVE 中如何扩容

进入：

**虚拟机 -> 硬件 -> 选中硬盘 -> 磁盘操作 -> 调整大小**

填写要新增的容量，例如：

- 推荐先增加 **16G**

后续如果不够，可以继续扩容。

### 注意事项

当前 **不支持热扩容立即生效**。

也就是说：

- 在 PVE 里调整硬盘大小后
- 需要 **重启虚拟机**
- 扩容才会在下一次启动时生效

---

## 八、常见问题

### 1. 为什么我在 PVE 里通过 URL 导入失败？

请先确认目标存储已经开启：

- `导入`
- `磁盘镜像`

路径：

**数据中心 -> 存储 -> 对应存储对象（例如 `local`） -> 编辑 -> 内容**

### 2. 出现以下错误怎么办？

```text
sata0: import working storage 'local' does not support 'images' content type or is not filebased
```

这通常表示：

- 你选中的 working storage 没有开启 `磁盘镜像`
- 或者它不是适合该导入流程的 file-based storage

处理方法：

1. 进入  
   **数据中心 -> 存储 -> local -> 编辑**
2. 确认“内容”里已勾选：
   - `导入`
   - `磁盘镜像`
3. 如果仍失败，请改用支持 file-based 的目录型存储，或者改走手动导入 `.img` 方式

### 3. 为什么启动后 `ip a` 里 `eth0` 没有分配 IP？

优先检查：

- 虚拟机是否有多张网卡
- 多张网卡的类型是否一致

如果存在混用，例如：

- 一张 `E1000`
- 一张 `VirtIO`

就可能导致：

- `eth0` / `eth1` 顺序混乱
- `eth0` 没有分配到预期 IP

解决方法：

1. 把所有网卡改成一致类型
2. 重启虚拟机
3. 再次执行 `ip a` 检查

### 4. OVA 导入后还需要检查什么？

建议至少检查：

- 启动模式
- 磁盘控制器
- 网桥绑定
- CPU 类型
- 网卡类型是否一致

如果是老 CPU，建议优先手动设置 CPU type 为 `host`。

### 5. 导入后为什么磁盘容量没有立刻变大？

因为当前扩容是在**启动时生效**的。

如果你刚在 PVE 中执行了“调整大小”：

- 需要重启虚拟机
- 才会在下一次启动时自动扩容
